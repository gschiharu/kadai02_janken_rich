## ⑤工夫した点・こだわった点
-はじめ、じゃんけんアプリと同じロジックで、サービスに対して担当者はコンピューターが選ぶようにしましたが、実際には、担当者がきまっているので、違った関数にしてみたところです。
最終的につくりたいサービスアプリのイメージをして、本当のさわりの部分を今の情報でできるところをやってみました。（まだまだ内容はすっからかんですが！）


## ⑥難しかった点・次回トライしたいこと(又は機能)

-本当は、サービス提供側のメンバーが10名前後いて、それぞれのタイムスケジュールを連動したアプリをつくりたいと思いました。将来的には、スケジュールアプリ（Calendly）と決済ツール（Stripe）を連動させた内容に発展させていきたいです。

- [疑問]今回は、３つのサービスに3名の担当者がいる想定でしたが、実際には、各サービスに対して、2名以上可能な人がいて、”スキル”や”対応可能日時”にて推薦順位が決まってくる仕様にする場合はどんな仕組みが必要になるか知りたいです。
